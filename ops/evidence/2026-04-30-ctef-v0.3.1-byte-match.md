# CTEF v0.3.1 byte-match — `@foxbook/core` canonicalJcsBytes vs published vectors

**Date:** 2026-04-30
**Spec:** CTEF (Composable Trust Evidence Format) v0.3.1
**Vectors source:** [`https://agentgraph.co/.well-known/cte-test-vectors.json`](https://agentgraph.co/.well-known/cte-test-vectors.json)
**Canonicalization:** RFC 8785 (JSON Canonicalization Scheme)
**Hash algorithm:** SHA-256
**Outcome:** **4/4 vectors byte-match** — canonical bytes byte-exact, SHA-256 hex-exact

## Implementation

Foxbook's RFC 8785 canonicalizer is `canonicalJcsBytes(obj: unknown): Uint8Array` exported from
[`core/src/crypto/jcs.ts`](../../core/src/crypto/jcs.ts), a thin wrapper around
[`canonicalize` v2.1.0](https://github.com/erdtman/canonicalize) — the reference
implementation by Anders Rundgren, who also authored RFC 8785.

```typescript
// core/src/crypto/jcs.ts (excerpt)
import canonicalize from "canonicalize";

const textEncoder = new TextEncoder();

export function canonicalJcsBytes(obj: unknown): Uint8Array {
  const canonical = canonicalize(obj);
  if (canonical === undefined) {
    throw new TypeError("…");
  }
  return textEncoder.encode(canonical);
}
```

## Test harness

[`core/__tests__/canonical-jcs.test.ts`](../../core/__tests__/canonical-jcs.test.ts) runs in two modes:

- **Offline** (always in CI): reads the fixture at [`core/__tests__/fixtures/ctef-v0.3.1.json`](../../core/__tests__/fixtures/ctef-v0.3.1.json) (snapshot from 2026-04-30T10:51Z) and asserts both `canonicalJcsBytes(input_object)` decoded as UTF-8 equals `canonical_bytes_utf8` AND `SHA-256(canonicalJcsBytes(input_object))` equals `canonical_sha256` for all four vectors.
- **Live** (`RUN_LIVE_TESTS=1`): re-fetches `cte-test-vectors.json`, asserts the published SHA-256s match the fixture (drift detector), then re-asserts our canonicalizer byte-matches against the live document.

Both passed at 2026-04-30T10:32Z.

**Note on the negative-shape vectors (3 + 4):** the byte-match test is pure RFC 8785 canonicalization — `canonicalize(input_object) → SHA-256` matches the published hash. Whether a downstream CTEF verifier *rejects* these inputs with `INVALID_CLAIM_SCOPE` / `INVALID_COMPOSITION` is a layer above canonicalization (semantic validation) and is out of scope for this report. Foxbook is the verification primitive underneath; it ships the canonicalization correctness, not the CTEF claim validators.

## Reproduce

From the foxbook repo root:

```bash
pnpm install --filter @foxbook/core
pnpm --filter @foxbook/core test canonical-jcs
```

To re-run against the live document (network required):

```bash
RUN_LIVE_TESTS=1 pnpm --filter @foxbook/core test canonical-jcs
```

## What this validates

Foxbook's `canonicalJcsBytes` produces byte-identical output to the agentgraph-co/agentgraph reference (`src.signing.canonicalize_jcs_strict`) on every published CTEF v0.3.1 vector. Anyone running CTEF verification against attestations Foxbook signs (or vice versa) is interoperating against the same canonical-bytes contract that the rest of the implementations in the [A2A #1672 convergence harness](https://github.com/a2aproject/A2A/discussions/1672) implement.

This report is the witness. The fixture pins the bytes; the test pins the canonicalizer; the SHA-256 results pin both.
