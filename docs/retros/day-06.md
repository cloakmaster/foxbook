**Now known that wasn't.** (1) **Drizzle 0.45.x wraps postgres-js errors in `DrizzleQueryError`** with the original SQLSTATE on `e.cause.code`, not `e.code`. The repository's catch in PR #14 (Day-5) inspected only the top-level — so the 23505 unique-violation fell through and Hono surfaced a generic 500. The smoke script's hardcoded "Migration 0001 not applied" hint compounded the misdirection. PR #28's fix is a `walkCauseChain` helper; the lesson generalizes — any error-classification logic against an ORM's wrapped errors needs cause-chain inspection, not single-level `code` comparison. (2) **vitest's default invocation doesn't pick up Node's `--env-file=` flag** because the `apps/api` `test` script runs `vitest run` directly, not through `node --env-file=...`. Smoke scripts and the dev server use `node --env-file=...`; vitest has its own loader that bypasses Node's startup env hooks. PR #32's localized `loadEnvFile(.env.local)` call inside the integration test file is the minimal fix; the more robust path is a `vitest.config.ts` with a `setupFiles` hook, but that adds config-file surface for one test, so localizing won. (3) **`apps/api` dev is `node --import tsx`, not `tsx watch`** — the dev server doesn't auto-reload on file changes, which bit twice today (PR #28's Drizzle-error fix didn't take effect until restart; PR #29's `/claim/revoke` route didn't mount until restart). Day-7 housekeeping ticket: switch to `tsx watch` in the `dev` script + add `apps/api/scripts/dev-restart.sh` that wraps `kill $(lsof -t -i:8787) && pnpm --filter @foxbook/api dev` so the kill-and-restart dance is one command.
