**Bench measured.** `core/scripts/bench-merkle-append.mjs` @ 10K iters on darwin/arm64 / Node v25.2.1: **p50 0.0112 ms, p95 0.0155 ms, p99 0.0221 ms**, ~87 K appends/sec single-threaded, no DB. That's three orders of magnitude below the 50 ms-at-100 rps end-to-end SLA (`PROJECT-PLAN.md` Day 4 gate / foundation §11.1), so **TS tree math is not the bottleneck** and the Go-daemon escalation in foundation §11.2 does **not** fire on this evidence. **Now known that wasn't.** (1) RFC 9162 §2.1.4.3 consistency-proof verification is the fiddliest bit — the "re-prepend `old_root` when `m` is a power of 2" detail is what separates a working implementation from one that silently accepts invalid proofs; it's now anchored by 45 committed vectors across every (m, n) with 0 ≤ m ≤ n ≤ 8 in both TS and Python. (2) `canonicalJsonBytes` extraction confirmed that insertion-order `JSON.stringify` ↔ `json.dumps(separators=(',', ':'), ensure_ascii=False)` stays byte-identical for the STH payload shape `{log_id, tree_size, root_hash, timestamp, version}` — the existing `jws_round_trip` vector is our continuous parity alarm, so we don't need RFC 8785 JCS in v1. (3) AJV 8 + NodeNext + draft 2020-12 has real interop friction (the `Ajv2020` class ships without an explicit `exports.default`, so named-imports are required and the ajv-formats/ajv-errors defaults need a typed-cast at the seam); documented in-file in `packages/validators/src/index.ts`. (4) `json-schema-to-typescript` and `quicktype` both try to resolve external `$ref` URLs over HTTP by default — `scripts/generate-types.mjs` now carries a local HTTP resolver that maps `https://foxbook.dev/schemas/*` to committed files, and cross-source registration is conditional on substring `$id` match so standalone `envelope.py` / `discover_response.py` output stays byte-stable across regenerations.
